1. Getting Started

Hi! I'm Idra Hsieh -- a developer who believes good software is built with both logic and empathy.

I started learning web development because I wanted to understand how ideas become experiences — how design and code come together to make something that feels effortless to use.

2. Folder Structure

I like keeping my project structure minimal and easy to read. Each file has its own clear responsibility — this helps me think more clearly when building or debugging.


                project/
                ├─ index.html
                ├─ styles.css
                ├─ script.js
                └─ assets/
                ├─ icons/
                └─ images/
  • index.html - structure and semantics
  • styles.css - layout and visual design
  • script.js - logic and interactivity

Starting simple gives room to grow without losing direction. I often refactor as I learn new patterns — clean structure supports clean thinking.

3. Design System

Design systems help me stay consistent and make small visual changes with confidence. I use CSS variables to define colors, spacing, and rounded corners across projects.


                :root {
                --bg: #f6f1e9;
                --ink: #1f1b16;
                --accent: #b07a3f;
                --radius: 14px;
                }

I value soft tones, clear typography, and breathing space. I want users to feel calm, not overwhelmed.

  • Keep layouts simple and accessible
  • Use whitespace to focus attention
  • Let motion guide, not distract

Good design isn’t just how it looks — it’s how comfortable it feels to use.

4. Coding Habits

I try to write code that future me — or anyone else — can easily understand. Clarity always comes before cleverness.


                function greet(name = "World") {
                    return `Hello, ${name}!`;
                }

                console.log(greet("Idra"));
  1. Use clear, action-based names for functions and variables
  2. Write small, focused components
  3. Organize logic before styling
  4. Keep comments meaningful and concise
  5. Commit often with descriptive messages

To me, good code feels steady — it reads like a story of how things work together.

5. DOM and Interaction

Interactivity is where users meet the code. I enjoy building small, thoughtful interactions that respond naturally to input.


                const input = document.getElementById("text-input");
                input.addEventListener("input", () => {
                console.log("User is typing...");
                });
  • Ensure buttons and forms are keyboard-friendly
  • Maintain visible focus indicators
  • Provide instant feedback for user actions
  • Keep motion subtle and purposeful

I believe the best interfaces guide users gently. Accessibility isn’t optional — it’s part of good craftsmanship.

Deployment and Beyond

Deployment is one of my favorite steps — it’s when an idea finally becomes something real and shareable. I use platforms like Vercel for quick and reliable hosting.


                git add .
                git commit -m "deploy: first live version"
                git push origin main

After deploying, I always check how the page feels — speed, readability, and flow matter as much as function.

  • Measure load times and accessibility scores
  • Test across screen sizes and devices
  • Iterate often, keeping quality steady

My goal is to build things that feel polished, kind, and reliable — experiences that people trust.
